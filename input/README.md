收纳用户人工输入、原始输入资料、历史参数表和候选人简历等输入材料。

## 目录角色

用于保存设计前提、阶段性澄清、早期原始资料和团队候选人输入，供后续文档追溯引用。

## 子目录索引

- `00_requirements/README.md`：需求与参数输入。核心主题：用户需求事实源、早期硬件参数表与相关输入资料。
- `01_candidate_resume/README.md`：候选人简历输入。核心主题：候选人简历资料，以及与团队规划基线协同评估的使用方式。

## 维护规则

1. 文档内容继续使用中文撰写。
2. 文件名使用英文小写、数字前缀和生命周期目录组织。
3. 目录级变更统一只在根目录 `CHANGELOG.md` 中记录。
4. 当目录内新增文档时，需同步回写本 `README.md` 的目录索引。
5. `00_requirements/` 由用户主导持续补充需求与参数输入，其他线程只读取不改写核心事实源。
6. `01_candidate_resume/` 用于存放候选人资料；候选人评估线程应以 `docs/10_team_planning/01_development_team_proposal.md` 作为组织能力基线。
