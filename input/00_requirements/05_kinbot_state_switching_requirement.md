1. # 需求背景
    

- 明确机器人核心状态（关机、待机、工作、充电、故障）及其他补充状态的定义，规范各状态间的切换触发条件、操作路径、约束规则，确保机器人状态切换逻辑统一。
    

  

2. # 状态与切换需求描述
    

3. ## 功能概述
    

- 实现机器人多状态的规范化管理以及自动切换，通过对关机、待机、充电、交互等其他状态的精准定义和切换说明，保证机器人在不同任务与功能之间能实现准确状态切换。
    

2. ## 状态描述
    

|   |   |   |
|---|---|---|
|**状态**|**描述**|**典型行为**|
|**关机状态**|机器人完全断电，无法响应任何指令。|无动作，仅响应硬件开机键；能关机充电。|
|**待机状态**|系统启动，传感器就绪，未执行具体任务。|可被唤醒，检测环境声音，摄像头开启，屏幕显示待机页。|
|~~**休眠状态**~~|~~长时间无~~~~任务和~~~~交互时进入的低功耗模式。~~|~~可被唤醒，检测环境声音，摄像头开启，屏幕熄屏。~~|
|**交互状态**|正在与用户进行语音、触控和手势交互。|停留在用户前，或缓慢跟随，与用户进行语音、触控和手势交互。|
|**巡逻状态**|在家庭内进行自主导航巡检。|开启移动、避障、定点停留观测。|
|**找人状态**|接收指令寻找特定人员（基于人脸/声纹/特征）。|开启移动、避障，找到用户后开始交互。|
|**找物状态**|接收指令寻找特定物品。|调用家庭（时间&空间）记忆，语音提醒用户物品上次放置的位置（交互状态）；如未有历史记忆，开启移动、避障、视觉扫描环境，匹配物体特征，发现后标记位置并通知用户（找物状态）。|
|**返航状态**|任务结束或收到指令，返回充电桩。|规划最短路径回充，途中可被高优先级事件打断。|
|**常规低电状态**|电量低于10%。|提示“电量低”。若空闲则立即返航充电；若任务中则评估电量是否足够，足够则完成后返航充电，不够则提示用户并返航充电。|
|**深度低电状态**|电量低于5%。|最高优先级，中断当前任务，立即返航充电。|
|**充电状态**|已连接充电桩并进行充电。|静止充电，可被唤醒。|
|**故障状态**|检测到硬件或软件错误。|停止运动，声光报警，等待人工复位或自动恢复。|
|**急停状态**|物理急停按钮被按下。|立即切断动力输出，保持系统供电，需人工解除。|
|**升级状态**|正在进行固件或软件包更新。|停止所有运动，显示进度条，不执行其他任务。|
