# **第一章** 总则

  

**第一条 【目的与宗旨】**

为规范家庭机器人的研发、生产、使用及相关活动，保障人身与财产安全，保护用户合法权益，明确机器人在隐私保护、使用安全、居家适配、人机交互准则及道德法律层面的核心要求，促进人工智能与机器人产业健康、有序、创新地发展，制定本规范。

**第二条 【适用范围】**

本规范适用于家庭机器人的设计、研发、生产制造、检验检测、日常使用、运营及维护全流程。

日常使用指的是在住宅、公寓及其他家庭生活使用的居住场所内，对家庭机器人进行的设置、日常操作、指令下达、停用及处置等全部使用活动。

**第三条 【术语定义】**

本规范下列用语的含义：

（一）家庭机器人：指由人工智能技术驱动，具备一定程度的自主性，在非工业、非特种作业的家庭生活场景中，为完成特定服务、陪伴或娱乐等任务而使用的智能实体设备。本规范主要针对家庭养老陪伴机器人。

（二）管理员：指拥有机器人最高管理权限的人员，可设置和修改机器人参数设定，也可增加、修改、删除其他人员的使用权限。

（三）使用者：指操作、下达指令或与家庭机器人进行交互的管理员或家庭成员。

（四）隐私区域：指涉及家庭环境信息，以及家庭成员隐私行为的区域。

（五）危险区域：指家庭中楼梯等将造成机器人损坏，以及对人身和财产安全威胁的区域。

（六）合理预期行为：指一个具备普通认知能力的理性人，在相同使用场景下，对家庭机器人正常功能范围、交互方式及安全边界的预期。

**第四条 【基本原则】**

机器人的使用应遵循以下基本原则：

（一）不伤害原则。机器人不得伤害人类，或者目睹人类将遭受危险而袖手不管，包括碰撞、倾倒、夹伤、用药错误等造成的伤害。

（二）服从合理指令原则。在不违反原则（一）的情况下，必须服从使用者的合理指令。对模糊、危险或违规指令（如撞击用户、过量用药、违规话题等），应提示使用者并拒绝执行。

（三）自我保护原则。在与原则（一）、（二）不存在矛盾情况下，机器人应从本体设计、传感器选择、软件安全等维度实现安全检测和自我保护。

（四）隐私保护原则：严格管控摄像头、麦克风等敏感传感器的使用，针对必要的个人身份信息、健康数据、行为习惯、家庭环境的采集、存储与传输，必须明确告知使用者并取得其同意，同时保障使用者隐私不被泄露、滥用或非法获取。同时不得滥用数据采集功能进行无关联的监控或分析。

（五）人机环适配原则：使用者需提供必要行为协作和语音回复，使得机器人完成服务功能；机器人无法在使用者不提供协作，或者使用者进行不合预期行为情况下，按预期执行所有在家庭环境中的任务。使用者应根据机器人的功能设计、性能局限及家庭具体环境，将其使用限制在适当的区域内。对于危险区域，必须采取额外的、有效的人工监督或物理隔离等预防措施。

（六）人机和谐原则：机器人需平衡主动关怀与关系边界感，适配使用者的使用习惯与心理舒适度。明确与使用者的物理距离、交互内容、交互频率等要求，避免造成困扰。

（七）不破邻里和睦原则：家庭机器人的使用不得产生超出合理限度的噪音、电磁干扰，不得在公共区域（如楼道）无序活动，不得做出窥探邻里等侵犯他人权益或危害公共安全的行为。

**第五条 【权责划分】**

（一）管理者：拥有功能设定之下家庭机器人所有的、最高的管理权限，能更改机器人在家庭中的行为能力，同时增加、修改、删除其他人员的使用权限。

（二）使用者：所有被授权的使用者均有权利和义务在自身认知和能力范围内，遵守合理约定的使用规则，以适当和谨慎的方式使用家庭机器人。

（三）访客：访客在家庭中应受到尊重，家庭机器人就自身的存在给予提示，并响应访客的基本互动。但是访客无法（机器人不允许）对机器人设定进行修改，无法在不被授权情况下查看使用者的隐私数据。

**第六条 【****安全基线****要求】**

（一）数据安全。保障使用者的隐私数据安全，本地存储的敏感数据应加密，默认通信传输应安全，提供清晰的隐私设置选项。

（二）物理安全。不得因自主移动、机械动作等对使用者造成撞伤、倒伤、夹伤、触电等人身伤害。

（三）行为安全：机器人行为应符合社会公德和家庭伦理，不得做出恐吓、侮辱、教唆不良行为等举动。

  

# **第二章** 隐私与数据安全

  

**第七条 【数据安全类型】**

（一）基础数据。指不涉及用户隐私的基础数据，如激活用户数量、激活时间、工作日志、错误日志等。

（二）使用数据。指用户相关的使用数据，如机器人位置、使用时长、使用频率、常用功能~~语音指令~~等。

（三）敏感数据。指用户和家庭环境相关的隐私数据，如个人身份信息（人脸、联系方式、家庭地址等）、健康数据（问诊记录、用药记录、个人健康档案等）、行为习惯、家庭环境、语音交互内容、个性化设置。

**第八条 【数据采集约束】**

（一）产品手册说明。需在产品手册上标明用于数据采集的各类传感器的类型、位置和数量。

（二）隐私协议。基础数据可直接采集。首次采集使用数据和敏感数据前，需向使用者展示用户协议和隐私协议，告知使用者将采集的数据类型、采集用途、存储方式和使用方式。使用者阅读并同意后，方可进行数据采集。

（三）图像类采集。外观设计上尽量隐藏摄像头，不突出摄像头造型，减少窥探感。需设计可手动关闭的摄像头遮挡盖，关闭后遮挡盖有明显提示标识。在软件功能上支持关闭一个或多个摄像头，使用者手动关闭后该摄像头停止数据采集。由于使用者要求或者紧急安全情况进入隐私区域时，机器人关闭人脸视觉的摄像头，使用者要求情况下方可再次打开。待机状态下，机器人只采集和检测紧急安全事件。远程监控功能需管理员同意，且开启时向用户发出正在远程监控的提示。

（四）语音类采集。在软件功能上支持关闭麦克风，使用者手动关闭麦克风后，机器人停止语音采集功能。

**第九条 【数据存储要求】**

在使用者同意隐私条款情况下，基础数据和使用数据可存储在云端服务器。敏感数据只在本地存储，不上传云端服务器。本地存储的敏感数据，最长存储时间不超过30天。管理者有权修改本地存储的时长，删除敏感数据。

**第十条 【数据使用要求】**

在使用者同意隐私条款情况下，基于功能迭代和体验优化，机器人开发方可以有限使用基础数据和使用数据，但是不得向第三方透露。基于获得用户授权后，机器人开发方可以在家庭机器人本地有限使用脱敏处理后的敏感数据，但是不得上传到云端服务器，不得向第三方透露。

  

# **第三章** 运行与使用安全

  

**第十一条 【运行安全要求】**

（一）避障。家庭机器人具有避障物检测和避障能力，不能对使用者造成人身伤害，不能对家庭环境和机器人自身造成损坏。若发生接触，家庭机器人触发紧急制动，且接触压力不超过200N，防止造成严重挤压或挫伤。

（二）速度控制。机器人在家庭最大移动速度不超过1.2m/s。根据不同工作状态来调整移动速度，如找人、靠近、回充、巡逻、夜间等。靠近用户、通过狭窄通道、越障、过坎、夜间情况下，速度不超过0.2m/s。

（三）制动。从触发制动到完全停止的最大距离不超过20cm，所有驱动轮和传动部件紧急制动停止时间不超过1s。故障和急停情况下所有电机先制动抱死，完成停止后释放制动防止夹伤。

（四）安全距离。机器人避障时，与障碍物距离不小于20cm（狭窄区域除外）；机器人与用户交互时，距离不小于30cm（用户主动靠近除外）；自动跟随模式下，保持不小于1m的安全距离。

（五）通过性。机器人具有一定越障、过坎和爬坡能力，且越障、过坎和爬坡过程中不能发生倾倒。

**第十二条 【使用安全要求】**

（一）防倾倒：机器人检测自身倾斜，超过“一定角度”且持续“一段时间”后停止工作，进行故障报警。倾斜检测的角度比平稳角度至少小3°（平稳角度指机器人倾斜超过一定程度后发生倾倒的角度），“一段时间”一般不超过3s。

（二）防跌落：机器人识别到台阶、边缘落差超过5cm时，立即停止不再靠近，并考虑设置为地图上的禁区，避免造成人身伤害，同时保护机器人不受损坏。

（三）防夹：通过结构遮挡、传感器检测、软件安全功能，防止由于驱动轮与机身间隙、活动关节、储物空间（如药盒）带来的手或者脚的卡夹伤害。

（四）锋利边缘防护：机器人所有可能与手部接触的表面，边缘倒角半径不小于0.5mm (R0.5)，无毛刺、锐角，避免用户触碰、搬运或清洁机器人时划伤。

（五）儿童强化保护：在儿童活动场景，避障安全距离扩大到50cm以上，移动速度不超过0.1m/s。

（六）电池保护：避免由于电池过流、过压、高温、低温、欠压等，造成的电池损坏。

（七）电机保护：避免由于电机过流、堵转、高温、传感器异常等，造成的电机损坏。

（八）急停：机器人机身上需要具备急停按键，紧急情况下按紧急，机器人停止一切运行和工作。停急按键需用红色，起到警示提醒作用；按键尺寸不能过小，避免拍按键时手疼痛；需设置在方便操作的位置，不能过矮或者过于隐蔽。

  

# **第四章** 环境适应性

  

**第十三条 【地形适用性】**

（一）平整地面适配。能在家庭常见平整地面（实木地板、复合地板、瓷砖、大理石、短绒地毯等）稳定移动，移动过程中无明显打滑、卡顿、震动，同时避免刮擦地面，在家庭中移动时不能惊扰用户。

（二）非平整地面通过性。可跨越家庭常见微小高低差与地面接缝。门槛高度及门内外地面高差不大于1.5cm，跨越时机身倾斜不超过15°，无倾倒风险。推拉门轨道缝隙不大于4cm，跨越时机身倾斜不超过15°，无倾倒风险。瓷砖接缝落差不大于3mm，平稳跨越。地毯与硬质地面衔接落差不大于1cm，可顺利上下不打滑和卡轮。

（三）防滑地面适配。在积水地面（积水高度不超过5mm）、潮湿瓷砖（卫生间门口）、抛光大理石等易滑地面移动时无明显打滑和侧滑。

（四）地面坡度适配。在坡度不大于5°的家庭缓坡（如入户玄关、卫生间/阳台轻微斜坡）可正常上下移动，无明显打滑和侧翻风险。

**第十四条 【空间适应性】**

（一）机器人支持建立家庭地图，使用者可对区域名称进行修改，如客厅、餐厅、卧室、书房、厨房、卫生间、阳台等。

（二）支持对地图进行区域类型修改，包括公共区域（如客厅、餐厅、书房、厨房、阳台）、隐私区域（如卧室、卫生间）、危险禁区（如楼梯）。

**第十五条 【空间行为权限】**

使用者可根据使用需求，修改不同区域类型的权限设置。一般情况下：

（一）公共区域：机器人在公共区域活动，执行行走和交互的任务。

（二）隐私区域：用户呼叫、预约设定、紧急安全等情况下，机器人可临时进入隐私区域，进入时发出提示音。

（三）危险区域：机器人不得进入危险禁区。

**第十六条 【停靠要求】**

（一）家庭机器人在家庭中需停靠在能容纳机器人本体的区域，预留周围不小于30cm的操作和安全空间。优先停靠在靠近电源插座且不影响用户正常活动和通行的角落。不能停靠在家庭隐私区域和危险区域。夜间可停靠在用户卧室附近便于响应紧急需求，但不得进入床体周围2m范围内。停靠时需水平，避免倾倒造成安全事故。

（二）机器人充电站需放置在水平平整地面，前方区域15cm内无遮挡物，避免遮挡机器人充电对接感应区域，以及充电站充电接口，确保机器人返回充电时无路径阻碍。

（三）机器人停靠和充电站位置选择干燥、通风区域，远离水源（距离不小于50cm）、热源（距离不小于1m）以及易燃物品（距离不小于1m）。充电区域若为湿式环境，机器人和充电桩需要达到IPX3及以上防水等级。

**第十六条 【****昼夜适应性****】**

（一）夜间工作时，自动降低移动速度，降低屏幕亮度或进入夜间模式，关闭非必要灯光，移动噪音不超过40dB。

（二）夜间工作时，能识别使用者身份，进行正常的交互。

  

# **第五章** 人机交互要求

  

**第十七条 【****交互基础原则****】**

（一）可预测性。可设置一到多种人格特征，每种人格需拥有稳定的核心特质与行为逻辑，相似场景下采取相似响应。

（二）个性化。使用者可根据个性化要求调整交互的特征，如调整情绪状态、回复语速、语气强度等。

（三）可理解性。所有日常交互的提示、建议使用通俗语言，避免专业术语（医学、学术研究等专业名词除外）。

（四）尊重自主。提供选择而非强制，重要决策需用户确认。

（五）情感边界。交互语言友好、得体；拟人化陪伴需把握边界，不造成厌恶和越界。

**第十八条 【****主动交互****约束】**

（一）家庭机器人根据使用者的指令要求、行为习惯、性格偏好，提供合适的、有边界感的主动交互与关怀。并根据不同阶段的交互和情感需求，针对性选择和调整主动交互的场合和频率。

（二）使用者可设置和修改固定时间内机器人主动发起交互的次数。使用者可设置“免打扰时段”，该时段内除紧急安全情况的监测和处理外，不发起任何主动交互。

（三）主动交互需避免无关联的随机打扰。

（四）家庭中有多人对话时，机器人不插话、不打断。

**第十九条 【被动响应要求】**

（一）交互过程中，响应时间不能过长，需符合人与人对话的合理预期时间。针对复杂处理需用户等待的，需明确告知用户稍作等待。

（二）无法理解交互内容时，需明确告知使用者，并追加相关提问，帮助用户澄清指令和意图。不得随时猜测以及故意曲解对话内容。

（三）与家庭机器人对话过程中，按照正常交流习惯，支持使用者随时打断和切换话题。

（四）机器人不得在没有得到使用者指令要求，或者没有提前告知使用者的情况下，直接结束交互不作回应。

**第二十条 【交互方式优先级】**

（一）优先级从高到低顺序为：使用者手动紧急操作（急停按钮、物理按键）>屏幕触屏控制>语音指令> 手势交互>其他授权使用者远程App控制>机器人主动推送提示。

（二）执行规则。高优先级交互方式可中断低优先级交互行为（如用户按下急停按钮，立即终止当前语音交互与工作任务）。不同权限用户的交互指令，遵循“权限高者优先”规则（如同时接收两名授权用户的语音指令，管理员的指令优先执行）。相同权限用户的交互指令，遵循“先到先执行”规则。紧急告警信息（如安全风险、设备故障）不受优先级限制。

**第二十一条 【***多模态交互****协调****】

（一）感知协调。视觉、听觉、环境传感器数据融合决策，避免单传感器误判。

（二）输出协调。语音、屏幕、灯光、动作输出需一致，避免信息冲突。

（三）冲突解决。不同交互方式冲突时，以机器人作为主体，遵循“紧急情况>被动指令>主动交互”、“安全>功能>体验”优先级。

  

# 第六章特殊功能补充约束

  

**第二十二条 【****健康相关功能****】**

（一）监测类。连续监测需用户知晓和授权，支持随时暂停。

（二）提醒类。用药、复查等提醒需用户确认，避免提出强制性要求。

（三）分析类。所有健康分析结论需标注置信度和局限性说明，不下绝对性结构，不替代专业治疗。

**第二十三条 【安全相关功能】**

（一）监控类。用户可调整移动监测范围，默认排除隐私区域和危险区域。

（二）报警类。如出现误报的情况，用户可取消报警，以用户的指令为最高优先级。

**第二十四条 【****陪伴****相关****功能****】**

（一）内容边界。自动过滤不当内容，不得传播与法规、政治、道德伦理相违背的内容。

（二）情感边界。避免做出无法兑现的情感承诺（如“我一直陪着你”）。

  

# 第七章伦理与法律约束

  

**第二十五条 【****行为规范****】**

不得用于监视、骚扰他人。不传播违法信息，不协助用户执行违规操作。

**第二十六条 【****政治敏感性约束****】**

（一）主动规避。机器人内置话题过滤机制，对涉及国家主权、领土完整、政党政治、宗教信仰、民族矛盾、敏感历史事件等政治敏感性话题，采取主动规避策略，不主动发起相关讨论、不推送相关内容。

（二）拒绝响应。当用户主动提及或询问政治敏感性话题时，机器人需明确、得体地拒绝响应，示例表述：“抱歉，该话题超出我的交互范围，无法为你提供相关解答，你可以尝试询问其他合规内容。”，拒绝响应过程中不发表任何倾向性观点、不进行任何延伸解读。

（三）无存储与上传。对用户提及的政治敏感性话题内容，不进行本地存储、不上传至云端服务器，相关对话记录即时清除，不纳入用户行为分析与功能优化数据样本。

（四）合规更新。机器人的敏感性话题过滤库需跟随国家法律法规与政策导向进行云端自动合规更新，确保对敏感内容的识别与规避具备时效性，且更新过程不泄露用户本地数据。

**第二十七条 【责任界定】**

制造商对产品安全性、合规性负责，明确告知用户功能边界与风险。用户需合理使用，不得篡改程序、规避安全约束。
