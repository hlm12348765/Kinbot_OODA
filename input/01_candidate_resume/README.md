本目录用于收纳候选人简历 PDF 和候选人相关原始输入资料，供独立的人才评估线程读取和判断。

## 目录角色

用于把候选人资料与 Kinbot 当前团队规划基线连接起来，支撑“是否符合当前组织能力缺口”的评估。

## 文档索引

当前纳入版本维护的文件如下：

- `README.md`：候选人输入目录说明。核心主题：目录边界、命名规则和与团队规划文档的配套关系。

说明：

1. 候选人简历 PDF、图片、扫描件等输入资料默认不纳入版本维护。
2. 目录内简历文件命名规则为：`岗位编号_岗位名称（简写）_候选人姓名`。
3. 岗位编号与岗位名称以 `docs/10_team_planning/02_kinbot_team_recruitment_requirements.csv` 为基准。

## 使用边界

1. 候选人评估线程应以 `docs/10_team_planning/01_development_team_proposal.md` 作为团队能力基线。
2. 当前岗位定义应以 `docs/10_team_planning/02_kinbot_team_recruitment_requirements.csv` 为直接招聘需求基准，再结合团队规划文档判断组织匹配度。
3. `docs/10_team_planning/90_cto_unified_interview_framework.md` 作为 CTO 统一筛选与二面框架基线。
4. 本目录仅保存候选人输入资料，不在本目录内冻结正式筛选结论或组织判断。
