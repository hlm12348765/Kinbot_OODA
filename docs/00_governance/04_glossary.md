## 1. 目的

本文件用于统一项目术语，避免同一名词在不同文档里含义漂移。

规则：

- 尽量遵守智驾、机器人、具身智能和软件工程里已有的约定俗成用法
- 如果项目必须引入自定义术语，必须同时给出清晰定义
- 同一术语在不同文档中不得出现互相冲突的含义

## 2. 核心术语

### `OODA`

定义：

- `Observe / Orient / Decide / Act` 的闭环方法论，用于组织机器人在动态环境中的感知、理解、决策和执行。

说明：

- 本项目中的 `OODA` 不是固定单轮控制环，而是多尺度、并发、可中断、可动态调度的正式架构基线。
- `Predict` 与 `Learn` 不作为 `OODA` 主环上的固定串行步骤，而是作为跨环能力贯穿 `Orient / Decide` 与跨周期演进过程。

### `Predict`

定义：

- 机器人对环境变化、任务结果、风险扩散和人类反应进行前瞻判断的跨环能力。

说明：

- 在本项目中，`Predict` 主要嵌入 `Orient / Decide`，而不是被机械追加到 `OODA` 主环尾部。

### `Learn`

定义：

- 机器人跨任务、跨周期、跨版本沉淀经验并改进未来表现的跨环能力。

说明：

- 在本项目中，`Learn` 既包括研发阶段的真值链与数据飞轮，也包括未来在合规边界内建立的受控经验学习。

### `Agent 增强平面`

定义：

- 在不新增一级模块的前提下，对 Kinbot 既有模块进行增强的一组跨模块能力平面。

说明：

- 一代正式范围包含长期记忆、技能化能力、连接器抽象与受控任务编排。
- 它增强 `Orient / Decide / 服务升级`，但不替代 `OODA` 主环。

### `技能化能力`

定义：

- 把机器人能力组织成可识别、可组合、可授权、可审计的技能目录，而不是散落在各处的逻辑片段。

说明：

- 一代主线只正式承接固定技能与组合技能。
- 候选新技能可进入研究线，但不自动进入正式能力集。

### `连接器抽象`

定义：

- 面向手机、App、云服务、智能家居和第三方平台的统一接入边界。

说明：

- 它的目标是避免服务接入继续退化为点对点耦合。
- 在本项目中，连接器抽象是伴生系统与服务扩展的长期稳定边界。

### `受控任务编排`

定义：

- 在安全、合规、授权和审计边界内，对技能、连接器和服务升级进行有约束的任务组合与调用。

说明：

- 一代允许受控任务编排，但不允许开放式自主代理。

### `自主创造新技能`

定义：

- 由模型或代理自动生成新的技能定义、执行链和接入逻辑，并尝试纳入运行时能力集的能力。

说明：

- 当前项目将其定义为研究线，而不是一代正式承诺。
- 即使后续探索，也必须经过评审、治理和审计边界。

### `R1 / R2 / R3 / R4`

定义：

- Kinbot 当前多尺度 `OODA` 的四类子环。

说明：

- `R1`：反射与安全环，强调毫秒到百毫秒级响应。
- `R2`：执行与交互环，强调亚秒到数秒级闭环。
- `R3`：任务与照护环，强调数秒到分钟级任务组织。
- `R4`：关系与服务环，强调小时到天级的长期关系与服务协同。

### `OODA Scale Scheduler`

定义：

- 决定当前由哪类 `OODA` 子环主导、以及不同子环如何并发与抢占的一级架构能力。

说明：

- 它不是普通任务队列，而是连接时延、算力、风险、用户上下文和运行模式的调度中枢。

### `家庭元场景`

定义：

- 家庭机器人长期面对的共性场景集合，而非单个具体房间或单次任务。

包含：

- 空间变化
- 光照变化
- 家庭成员变化
- 任务碎片化
- 陪伴、健康、安全、服务的长期叠加

说明：

- 本项目讨论“家庭元场景”时，强调的是长期稳定提供价值的系统能力，而不是某一个 Demo 场景。

### `同代际优化`

定义：

- 在同一套家庭共存样式下，对能力、成本、效率、时延、可靠性和产品感的持续强化。

说明：

- 更大算力、更低 `BOM`、更稳纯视觉、更强交互，通常都属于同代际优化，而不自动构成代际跃迁。

### `代际跃迁`

定义：

- 机器人与人、家庭空间、服务系统之间的基本协作样式发生变化。

说明：

- 在本项目中，代际跃迁不以器件升级为主要判据，而以“下一代人机共存的家庭生活”是否被定义出来为核心判据。

### `信息服务`

定义：

- 机器人主要改变信息流、认知流和决策流，而不稳定改变家庭物理环境的服务形态。

示例：

- 问答
- 提醒
- 监测
- 建议

### `物理服务`

定义：

- 机器人通过移动、递送、位置改变、空间介入或更复杂操作，稳定改变物体与家庭物理环境状态的服务形态。

说明：

- 在本项目中，从信息服务迈向物理服务，被视为重要的代际分界线之一。

### `ConOps`

定义：

- `Concept of Operations`，即系统在真实使用场景中如何发挥作用的运行概念描述。

说明：

- 在本项目中，`ConOps` 主要回答“谁在使用系统、系统通过哪些任务线程提供价值、在家庭中如何持续运行”。

### `OpsCon`

定义：

- `Operational Concept`，即系统由哪些运行节点、在哪些运行边界内被实际交付和协同的描述。

说明：

- 在本项目中，`OpsCon` 主要回答“价值由本体、伴生系统、人工服务、外部生态中的哪些节点交付出来，以及它们如何协同”。

### `PDCP`

定义：

- 产品定义与架构冻结阶段的系统级评审节点。

说明：

- 在本项目中，`PDCP` 不是量产评审，而是“完整系统架构基线是否成立、能否下发总体方案与模块方案”的关键阶段门。

### `Body Capability Contract`

定义：

- 连接机器人本体实体域与运行时模块的一级能力接口契约。

说明：

- 它不是器件清单，而是对本体可提供哪些稳定能力边界的正式表达，用于约束软件、算法、结构和整机方案的一致性。

### `World State`

定义：

- 机器人当前用于决策、执行和审计的统一结构化状态平面

包含：

- 当前人、物、空间、任务、风险、权限、健康和外部联动状态

不包含：

- 对未来环境动态的生成式预测能力

说明：

- 本项目当前文档中的认知中枢统一称为 `World State` 或 `World State Store`

### `World Model`

定义：

- 用于预测、生成、模拟或想象未来环境、任务或交互动态的模型

典型用途：

- 反事实推演
- 模拟与仿真
- 未来状态预测
- 规划前瞻

说明：

- 如果后续项目引入这类能力，可在 `Orient` 层作为可选预测模块加入
- `World Model` 不是当前“世界状态结构”的同义词

### `World Event`

定义：

- 从感知、任务或外部服务进入系统的结构化事件

### `DecisionContextSnapshot`

定义：

- 给一次具体决策使用的运行时快照

说明：

- 它是从 `World State` 投影出来的当前视图，而不是长期数据库

### `RoleBinding`

定义：

- 某个自然人在当前家庭中的角色和权限绑定关系

示例：

- 老人本人
- 子女
- 保姆
- 访客

### `Caregiver Mode`

定义：

- 面向保姆或照护者的协同工作模式

说明：

- 它不是一个人，而是一组针对 `caregiver` 角色启用的任务权限模板和交互策略

### `MedicationAsset`

定义：

- 被系统跟踪的药物实体，包括药品本身、存储位置、数量、有效期、处方引用和紧急使用规则

### `ServiceLink`

定义：

- 机器人与家庭外部服务主体之间的结构化联动对象

示例：

- 家属
- 社区
- 物业
- 医生平台
- 互联网医院
- 药店
- 配送服务

### `关系质量`

定义：

- 用于评价机器人与用户之间长期照护关系是否健康、可信、舒适和可持续的一组评价维度。

当前建议维度：

- 信任感
- 舒适感
- 克制感
- 连续性
- 可解释性
- 可恢复性

说明：

- 在本项目当前阶段，`关系质量` 首先是跨模块评价框架，而不是新的唯一运行时中心引擎。

### `L1 / L2 / L3`

定义：

- Kinbot 一代能力分层。

说明：

- `L1`：核心闭环能力，必须端侧闭环、断网可用、量产必须稳定。
- `L2`：增强认知能力，条件触发、按需启用，是技术突破重点。
- `L3`：服务扩展能力，依赖 App、云、人工服务和第三方生态，按“最小可交付 + 可收缩覆盖”设计。

### `真值参考链`

定义：

- 用于对主线算法或主线传感路线进行对照验证、误差分析和能力标定的研发验证链路。

说明：

- 在本项目中，深度相机和激光雷达属于纯视觉主线的研发对比基线与真值参考链，而不是产品级 fallback。

### `受控回流`

定义：

- 在明确授权、脱敏、加密、时效限制和用途限制前提下，对部分数据进行受限回传的能力预留。

说明：

- 本项目默认不把原始敏感数据回流作为一代主基线；`受控回流` 只表示预留、受限、可审计的能力。

## 3. 模型与性能术语

### `VLM`

定义：

- `Vision-Language Model`，即视觉-语言模型。

说明：

- 在本项目中，`VLM` 主要承担更高层的语义理解、图像问答、场景理解和复杂认知增强，不等于所有视觉任务都由 `VLM` 完成。

### `VLN`

定义：

- `Vision-Language Navigation`，即视觉-语言导航。

说明：

- 在本项目中，`VLN` 更适合作为高层语义导航与任务理解能力，而不是直接替代底盘级安全控制和局部运动控制。

### `TTFT`

定义：

- `Time To First Token`，即模型从接收请求到输出第一个 token 的时延。

说明：

- 本项目用它衡量端侧多模态模型在交互与任务触发中的起始响应速度。

### `TPS`

定义：

- `Tokens Per Second`，即模型单位时间内生成 token 的速度。

说明：

- 在本项目中，`TPS` 既包括常规输出速度，也常结合热稳态持续能力一起讨论。

### `热稳态持续 TPS`

定义：

- 在热约束和长时间运行条件下，模型可稳定维持的 `TPS`。

说明：

- 它比峰值 `TPS` 更接近真实产品体验，是端侧芯片路线评估的重要指标。

## 4. 研发与阶段术语

### `MVP`

定义：

- `Minimum Viable Product`，即最小可行产品。

说明：

- 在本项目中，`MVP` 不是“最少功能集合”的字面裁剪，而是围绕核心闭环、真实试点和可交付性组织的验证窗口。

### `EVT`

定义：

- `Engineering Verification Test`，即工程验证测试阶段。

说明：

- 在本项目中，`EVT` 是很多验证线、备线和研发参考链应退出或收敛的关键前置节点。

### `NPI`

定义：

- `New Product Introduction`，即新产品导入。

说明：

- 在本项目中，`NPI` 主要承接工程化、试产导入、制造验证与量产准备相关工作。
