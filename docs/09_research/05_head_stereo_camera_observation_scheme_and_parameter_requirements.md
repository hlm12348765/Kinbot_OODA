本文用于收敛 Kinbot 头部视觉观测方案中“主双目相机”的职责边界、参数计算方法和推荐指标。

本文默认继承以下前提：

- 一代传感主线为纯视觉。
- 机器人头部具备 `3DOF` 主动观察能力，可低头、抬头、转头和偏头。
- 头部主双目主要负责 `0.35-3m` 的主感知。
- 近距 `0.2-0.3m` 的精细识别不强行压在主双目上，建议由补充单目承担。
- 机器人头部安装高度约 `1.0-1.15m`。
- 机器人室内目标速度上限按 `1.2m/s` 评估。
- 双目链路默认与 `IMU` 融合使用。

本文是专项研究文档，不直接替代正式硬件冻结结论。正式冻结仍以 `P1 / PDCP` 主线文档和评审结果为准。

## 1. 机器人视觉观测方案设计思路

### 1.1 设计目标

Kinbot 头部视觉观测方案的目标，不是做一个“静态超大视场、尽量什么都看见”的相机，而是做一个“主动观察、前向主感知质量优先”的视觉系统。

因此，视觉观测方案应满足四个原则：

- 主双目优先解决 `0.35-3m` 主感知，而不是强行同时最优覆盖 `0.2m` 近距 OCR 和 `5m` 远距精细识别。
- 通过头部 `3DOF` 主动观察，减少对极端超广角镜头的依赖。
- 通过“主双目 + 近距单目 + IMU + 控速策略”的组合，替代“单一相机包打天下”的设计。
- 视觉数据除服务 VLN 外，还要兼顾状态估计、避障、交互、多模理解和视频通信等共享需求。

### 1.2 推荐观测架构

推荐的前视观测架构为：

- `1` 组头部主双目：负责 `0.35-3m` 主感知、状态估计、局部通行性判断、中距离人体与物体观察。
- `1` 个近距单目：负责 `0.2-0.3m` 的药盒、报告、OCR、小物体、近距细节复核。
- `IMU`：与主双目融合，用于提升运动状态估计、降低纯视觉短时失配风险。
- 头部 `3DOF`：承担主动补盲、低头看脚下、抬头看高位目标、转头看侧向目标的职责。

### 1.3 设计取舍

在头部有 `3DOF` 的前提下，主双目设计应从“极限覆盖”转向“高质量主视区”：

- 不优先追求 `150°+` 超大水平视场。
- 不把 `0.2m` 近距识别作为主双目第一目标。
- 不假设单靠 `30Hz` 双目就能兜底所有近场安全问题。
- 不把模型的理解与预测能力当成系统级安全门的替代品。

更合理的收敛方式是：

- 主双目看好前向主视区。
- 近距精细识别交给近距单目。
- 侧向与脚下补盲优先通过头部主动观察解决。
- 速度控制、局部规划和安全门控仍保留系统级约束。

### 1.4 职责分工建议

| 模块 | 主职责 | 推荐距离带 |
| --- | --- | --- |
| 头部主双目 | 双目几何、VO/VSLAM、局部通行性、门口/窄通道观察、中距人体/物体感知 | `0.35-3m` |
| 近距单目 | OCR、报告/药盒读取、小物体识别、近距复核、人脸/手势细节 | `0.2-0.5m` |
| 头部 `3DOF` | 主动补盲、视角调整、脚下确认、侧向确认、高位目标确认 | 全局补充 |
| `IMU` | 运动补偿、状态估计辅助、视觉退化时短时稳态支持 | 全局融合 |

## 2. 双目相机要解决的问题

### 2.1 几何感知问题

主双目首先要解决的是几何问题，而不是聊天式语义理解问题：

- 前向可通行区域判断
- 近中距离障碍与门框、桌腿、家具边界识别
- 窄通道、绕障、靠边通过时的几何稳定性
- 运动过程中的视觉里程计 / 视觉惯导融合

### 2.2 近中距离交互观察问题

主双目还需要兼顾近中距离的人机交互与观察：

- `0.5-1.5m` 的多模态交互
- 人脸、人体、手部与姿态粗识别
- 用户靠近、招手、转身、跟随对象确认
- 物体和人之间的空间关系理解

### 2.3 脚下与近端可见性问题

由于相机安装在头部 `1.0-1.15m` 高度，主双目必须回答一个工程问题：

- 在常规前视状态下，能否有效覆盖外壳前方约 `0.3m` 的地面与低矮障碍物。

这里不能只靠参数表判断，还必须结合：

- 安装高度
- 下俯角
- 外壳遮挡
- 头部运动策略

### 2.4 共享视觉栈问题

主双目不仅服务 VLN，还要服务整机其他功能：

- 语义导航中的前向观察
- 对话与陪伴中的人脸 / 姿态 / 注意力感知
- 家庭巡护中的人体和物体检测
- 视频通信中的区域裁剪
- 世界状态更新中的共享视觉输入

因此，主双目不能只按“研究 demo 最强精度”设计，还要兼顾：

- 长时间运行的热稳定性
- ISP / 带宽 / DDR / 算力预算
- 共享视觉栈下的复用效率

### 2.5 高速运动下的观测稳定性问题

在室内 `1.2m/s` 的上限速度下，主双目需要保证：

- 运动中不过度拖影
- 左右目时序严格一致
- 感知到控制链路的总延迟可控
- 在门口、转弯、近人、近障时支持降速与复核

这里的关键不是“只看帧率”，而是总链路：

- 采样频率
- 曝光时间
- 图像处理延迟
- 匹配 / 识别 / 状态更新延迟
- 规划与控制延迟

## 3. 参数计算公式

本节给出主双目参数选择时最常用的四类公式。

### 3.1 运动速度与帧率关系

机器人在相邻两帧之间的位移近似为：

```text
Δs = v / f
```

其中：

- `Δs`：每帧位移
- `v`：机器人速度
- `f`：相机帧率

当 `v = 1.2m/s`、`f = 30Hz` 时：

```text
Δs = 1.2 / 30 = 0.04m ≈ 4cm
```

这意味着：

- `30Hz` 对室内 `1.2m/s` 的主感知和状态估计是可行的。
- 但近场极限安全不能只靠 `30Hz` 双目本身兜底。

### 3.2 最近可见地面距离

当相机安装高度为 `h`，光轴下俯角为 `θ`，垂直视场角为 `VFOV` 时，地面最近可见点距离可近似表示为：

```text
d_near = h / tan(θ + VFOV / 2)
```

其中：

- `d_near`：最近可见地面距离
- `h`：相机中心离地高度
- `θ`：相机下俯角
- `VFOV / 2`：垂直半视场角

在本方案中，若 `VFOV = 100°`，则半视场角为 `50°`。

以 `h = 1.0m` 为例：

| 下俯角 `θ` | 最近可见地面距离 `d_near` |
| --- | --- |
| `15°` | `≈ 0.47m` |
| `20°` | `≈ 0.36m` |
| `25°` | `≈ 0.27m` |

这说明：

- 若头部高度约 `1.0-1.15m`，想覆盖外壳前方约 `0.3m` 的地面，通常需要约 `20°-25°` 的低头观察角度。
- 实际工程值还会受到前壳、面罩和镜头窗口遮挡影响，必须做 `CAD` 射线验证。

### 3.3 双目测距基本关系

双目测距的基本关系为：

```text
Z = f * B / d
```

其中：

- `Z`：目标距离
- `f`：等效焦距，单位通常为像素
- `B`：双目基线
- `d`：视差，单位通常为像素

这一定性说明：

- 基线越大，远距离视差越大，远距深度更有利。
- 焦距越大，视差越大，但视场会缩小。
- 视差越小，远距离深度越不稳定。

因此，`基线`、`视场`、`像素密度` 必须一起看，不能单看一个参数。

### 3.4 视场与等效焦距关系

若图像水平方向像素宽度为 `W`，水平视场角为 `HFOV`，则水平方向等效焦距可近似为：

```text
f_x = W / (2 * tan(HFOV / 2))
```

若图像竖直方向像素高度为 `H`，垂直视场角为 `VFOV`，则竖直方向等效焦距可近似为：

```text
f_y = H / (2 * tan(VFOV / 2))
```

这说明：

- 在分辨率固定时，视场越大，等效焦距越小，单位角度上的像素密度越低。
- 若希望兼顾 `3-5m` 的语义可用性，就不宜盲目把 `HFOV` 拉到极端超广角。

### 3.5 近似工程判断关系

对于主双目，可使用以下工程判断：

- `HFOV` 决定前向覆盖与像素密度之间的平衡。
- `VFOV` 决定近端地面覆盖能力与垂直像素密度之间的平衡。
- `基线` 决定 `0.35-3m` 与 `3-5m` 深度能力的平衡。
- `帧率` 与 `总延迟` 一起决定高速运动下的稳态表现。

因此，主双目参数选择本质上是四维折中：

```text
主双目质量 = FOV 覆盖 × 像素密度 × 基线几何能力 × 时序稳定性
```

## 4. 参数需求

### 4.1 主双目推荐参数

结合 Kinbot 当前目标、头部 `3DOF` 主动观察前提和室内 `1.2m/s` 速度约束，推荐主双目参数如下：

| 项目 | 推荐值 | 说明 |
| --- | --- | --- |
| 水平视场角 `HFOV` | `120°` | 接近人类双眼前向重叠主视区，兼顾覆盖与像素密度 |
| 垂直视场角 `VFOV` | `100°` | 配合低头策略与结构安装，可兼顾近端地面覆盖 |
| 分辨率 | `8MP` | 为 `3-5m` 中远距语义可用性和 `ROI` 裁剪留出余量 |
| 帧率 | `30Hz` | 可覆盖室内 `1.2m/s` 主感知；安全冗余依赖控速与门控 |
| 快门 | 全局快门 | 降低运动畸变，利于双目匹配、VO 与动态场景观测 |
| 曝光控制 | 主动曝光 + `HDR/WDR` | 适配室内逆光、强光、背光和频闪环境 |
| 时间同步 | 硬同步 | 左右目时序必须严格一致 |
| 标定 | 厂家标定 + 整机标定 | 量产需补装配后重标定与热漂检查 |
| 基线 | `7-8cm` | 兼顾 `0.35-3m` 主战场与安装、重叠、近距可用性 |

### 4.2 主双目的职责边界

主双目优先保证以下能力：

- `0.35-3m` 的双目几何主感知
- 前向可通行性判断
- `VO/VSLAM` 与 `VIO` 融合
- 门口、桌椅、窄通道、绕障等运动场景的稳定观察
- `0.5-3m` 的人体与物体中距离语义可用性

不建议把以下目标强行全部压给主双目：

- `0.2-0.3m` 的精细 `OCR`
- 贴身药盒、说明书、小字报告读取
- `5m` 内稳定高精度深度承诺
- 全场景无补盲高速近场安全兜底

### 4.3 近距单目补充参数建议

若前视允许增加一个近距单目，建议其参数方向如下：

| 项目 | 推荐值 | 说明 |
| --- | --- | --- |
| 分辨率 | `5-8MP` | 支撑近距小物体、OCR 与人脸细节 |
| 水平视场角 | `90°-110°` | 不必做极端超广角，优先保证像素密度 |
| 垂直视场角 | `70°-90°` | 服务近距桌面、胸前和手部区域 |
| 最近对焦距离 | 尽量做到 `10-15cm` | 保证近距可用性 |
| 帧率 | `30Hz` | 静态识别为主时足够；若兼顾手势可上探更高 |

推荐的职责边界是：

- 主双目负责 `0.35-3m`
- 近距单目兜住 `0.2-0.3m`
- 头部 `3DOF` 负责主动把目标带入合适观察视角

### 4.4 图像处理链建议

即便主双目传感器选用 `8MP`，也不建议默认整条链路都实时跑满分辨率双目。

更合理的方案是双链路并行：

- 低延迟几何链：降采样到 `2-4MP`，服务双目匹配、`VO/VIO`、局部通行性和近场动态更新。
- 高分语义链：保留 `8MP` 关键帧或 `ROI`，服务检测、识别、裁剪和复核。

这样做的理由是：

- 降低 `ISP`、带宽、`DDR`、算力和热设计压力。
- 保留高分辨率在关键任务中的收益。
- 更利于多功能共享视觉栈。

### 4.5 速度与控速建议

即便模型具备理解和预测能力，仍建议把速度控制分层：

| 场景 | 建议速度 |
| --- | --- |
| 开阔区域巡航 | `1.0-1.2m/s` |
| 常规家庭通行 | `0.6-0.9m/s` |
| 门口 / 窄通道 / 近障 / 近人 | `0.3-0.6m/s` |
| 起步 / 贴边 / 低头复核时 | 先降速或短停 |

模型的理解与预测能力应作为加分项，而不是系统级安全替代项。

### 4.6 工程验证重点

这组参数在冻结前，至少应验证以下内容：

1. `VFOV 100°` 在头部高度 `1.0-1.15m`、不同低头角度下，对外壳前 `0.3m` 地面的实际覆盖情况。
2. 前壳、面罩、装饰件对下边缘视线的遮挡。
3. `8MP + 全局快门 + 30Hz` 下的带宽、热设计、功耗和持续运行稳定性。
4. 室内背光、逆光、夜间弱光、`50/60Hz` 频闪下的左右目一致性。
5. `1.2m/s` 巡航时，`VIO`、检测、规划、控制的总延迟链路。
6. `0.35-3m` 中不同材质、黑色、反光、低纹理物体下的双目稳定性。

### 4.7 当前结论

在 Kinbot 当前约束下，更推荐的前视方案不是“一个超大视场双目解决全部问题”，而是：

- 用头部主双目解决 `0.35-3m` 主感知
- 用近距单目解决 `0.2-0.3m` 近距识别
- 用头部 `3DOF` 解决主动补盲
- 用 `IMU + 控速 + 局部安全门控` 解决高速运动与近场冗余

在这个前提下，主双目基线可收敛为：

- `HFOV 120°`
- `VFOV 100°`
- `8MP`
- `30Hz`
- 全局快门
- 主动曝光控制与高动态范围
- 硬时间同步
- `7-8cm` 基线

这组参数的定位应定义为：

- 适合作为 Kinbot 头部前向主双目基线
- 强项在 `0.35-3m` 主感知与中近距共享视觉任务
- 边界在 `0.2-0.3m` 精细识别和 `3-5m` 高精深度承诺
