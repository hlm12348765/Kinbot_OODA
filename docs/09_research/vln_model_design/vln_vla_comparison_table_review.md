### ⚠️ 语言地位（第 46 行）

```
| 语言地位 | 通常是核心目标载体 | 可能是条件之一，也可能在推理时弱化甚至省略 |
```

**准确性**：⚠️ 部分准确，但表述可能引起误解

**问题**：
- "可能在推理时弱化甚至省略"这个表述**过于绝对**
- 智驾 VLA 中，语言确实可以弱化（如 Wayve 的端到端驾驶），但也有很多工作保留语言作为重要输入
- 表述暗示"VLA 普遍弱化语言"，但实际上这取决于具体任务和设计选择

**建议修正**：
```
| 语言地位 | 通常是核心目标载体 | 智驾场景中常被弱化，但仍可作为重要条件输入 |
```

---

### ⚠️ 输出接口（第 47 行）

```
| 输出接口 | 离散动作、waypoint、子目标、轨迹块 | 连续动作、动作 token、统一策略输出 |
```

**准确性**：⚠️ 过度简化，存在误导

**问题**：
1. **VLN 领域并非统一输出子目标**：早期 VLN 工作（如 R2R）输出离散动作，近期工作有输出 waypoint 的，也有直接输出连续动作的
2. **VLA 领域也有输出离散动作的**：RT-1、RT-2 等机器人 VLA 输出离散动作 token
3. **这个区分混淆了"问题定义"和"实现选择"**

**建议**：删除此行，或改为"取决于具体方法设计"
