# Kinbot_OODA 软硬件选型矩阵

## 1. 文档目的

本文档用于回答 `KBT-11` 的核心问题：

在既定产品目标、成本约束和架构基线下，Kinbot 一代的软硬件路线应该如何收敛，哪些路线适合作为量产主线，哪些路线更适合作为 Alpha / EVT 验证加速路线，哪些路线只保留为前瞻观察。

## 2. 当前选型约束

本轮选型基于以下已确认约束：

1. 项目目标是在 `2026-12-31` 达到量产预备状态。
2. 整机物料成本目标为 `6000 到 8000 元人民币`。
3. 目标系统是机器人整机，穿戴、智能家居、App、云服务和后台运营属于伴生系统。
4. 原始视觉、语音和生物特征数据必须以端侧本地处理为原则。
5. 底盘倾向自研，麦克风阵列自研，穿戴外设不自研，相机优先供应商方案。
6. 当前样机已有粗放 Demo，但需要从验证平台收敛为量产平台。
7. `KBT-24` 已确认 `E4 伴生系统与服务运营基线` 优先，但伴生系统不能反向锁死本体硬件。

## 3. 当前选型原则

当前建议按 7 条原则收敛一代选型：

1. 优先服务量产，不为单次 Demo 继续堆配置。
2. 伴生系统优先冻结接口，不反向决定本体器件。
3. 对高风险器件保留 `A/B` 路线，避免被单点供应卡死。
4. 优先保留团队已有强项：底盘、麦阵、多模态交互、端侧算法。
5. 相机路线优先选择简单、可量产、可校准的方案。
6. 量产主线与验证加速路线可以并存，但必须在 `G2` 前收敛。
7. 对 2026 年仍快速变化的路线，只保留“前瞻观察位”，不轻易写入一代主线。

## 4. 七个选型域与当前判断

| 选型域 | 当前候选路线 | 当前判断 | 主要理由 | 当前风险 |
| --- | --- | --- | --- | --- |
| `S1 端侧算力平台` | `Qualcomm RB3 Gen2 / QCS6490`、`NVIDIA Jetson Orin NX`、`Rockchip RK3588`、`Jetson Thor/T5000` | 一代量产主线优先收敛 `RB3 Gen2/QCS6490` 与 `Orin NX`，`RK3588` 作为成本下探备选，`Thor/T5000` 仅保留前瞻观察 | 需要在生态成熟度、功耗、BOM、算法迁移代价之间平衡 | 若过早押注高算力路线，BOM/热设计压力会过大；若过早押注低算力路线，未来算法空间可能不足 |
| `S2 相机与深度路线` | 纯 `RGB` 多相机、自研深度估计；立体深度模块；结构光 / `ToF` | 量产主线优先 `RGB` 简配置 + 自研深度估计，立体深度模块作为 Alpha / EVT 验证加速路线 | 符合你对简单相机配置、成本和体积的要求 | 纯 `RGB` 路线对算法、标定和低光场景鲁棒性要求更高 |
| `S3 音频交互路线` | 自研麦克风阵列 + 端侧语音栈 | 维持自研主线，不做外采主路线 | 团队有明显既有能力，且这是产品体验核心资产 | 需要尽早冻结阵列结构、回声路径和整机声学布局 |
| `S4 底盘与运动执行路线` | 自研轮式底盘，传感器精简，自主运动控制 | 维持自研主线 | 机器人是目标系统，底盘体验直接决定一代可信度 | 若底盘接口与 `R1/R2` 运行栈不稳定，后续所有体验都受损 |
| `S5 储物仓、屏幕与整机结构路线` | 保留储物仓能力、重构传动机构；保留屏幕；整机外观形态重构 | 作为一代既定工程动作推进 | 这条链直接影响紧急给药、日常递送和整机感知价值 | 机构、重量、外观与装配空间容易互相牵制 |
| `S6 穿戴与家庭设备生态路线` | 手表 / 手环、蓝牙血压计、蓝牙血糖仪、智能家居设备 | 继续按“套餐推荐 + 生态接入”推进 | 这是首发健康场景的必要输入，不适合自研硬件扩张 | 协议碎片化、数据质量和接入一致性风险高 |
| `S7 软件栈与部署路线` | 端云协同、关键原始数据端侧处理、云侧承接知识与服务编排 | 继续按分层解耦路线推进，不提前锁死单一中间件 | 有利于在 2026 年快速变化的 AI 路线上保持灵活性 | 若接口口径不统一，后期迁移代价会很高 |

## 5. 端侧算力平台矩阵

### 5.1 候选路线

| 路线 | 当前定位 | 优势 | 风险 | 当前建议 |
| --- | --- | --- | --- | --- |
| `Qualcomm RB3 Gen2 / QCS6490` | 量产主线候选 | 面向机器人 / 边缘 AI，功耗和移动平台属性更友好，适合做成本和续航平衡 | 机器人生态与算法迁移便利性需继续验证 | 进入一代量产主线评估 |
| `NVIDIA Jetson Orin NX` | 量产主线候选 | 机器人与 AI 生态成熟，模型迁移和工具链相对友好 | 功耗、热设计和 BOM 压力更高 | 进入一代量产主线评估 |
| `Rockchip RK3588` | 成本下探备选 | 集成度高、成本和供应有优势，多媒体与多相机能力较强 | 算力与未来模型空间偏紧 | 作为成本型备选路线保留 |
| `Jetson Thor / T5000` | 前瞻观察 | 代表更强的机器人与生成式 AI 方向 | 对一代 BOM、功耗和周期压力过大 | 不进入一代主线，仅做前瞻观察 |

### 5.2 当前推荐口径

当前建议：

1. 一代量产主线只保留两条端侧算力候选：`RB3 Gen2/QCS6490` 与 `Orin NX`。
2. `RK3588` 作为成本下探与供应弹性备选保留，但不默认承载未来更重的本地模型路线。
3. `Thor/T5000` 不作为一代量产选型对象，只作为二代或平台前瞻观察位。

当前推断：

- 如果产品继续坚持“端侧处理原始感知 + 云侧承接通用知识与复杂服务”，一代未必需要追求极高端边缘算力。
- 如果一代希望在端侧承载更重的 `VLM/VLN` 推理，`Orin NX` 会更稳，但 BOM 和热设计压力也会更高。

## 6. 感知与交互器件路线矩阵

### 6.1 相机与深度

当前建议：

1. 量产主线优先 `RGB` 简配置多相机方案。
2. 深度能力优先通过自研估计模型和几何融合补齐。
3. Alpha / EVT 阶段允许引入立体深度模块缩短验证周期，但不默认进入量产主线。

### 6.2 麦克风阵列

当前建议：

1. 继续保持自研麦阵主线。
2. 尽快把阵列形态、波束策略、AEC 链路和整机声学设计纳入同一张工程表。
3. 不把语音体验问题推给云侧兜底。

### 6.3 穿戴与家庭设备

当前建议：

1. 继续按“推荐套餐 + 标准协议接入”推进。
2. 穿戴与家庭设备只通过稳定接口约束本体，不反向决定机器人主控路线。
3. 先保“高价值、低接入成本”的设备类型，再扩生态。

## 7. 当前推荐的主线 / 备线 / 观察线

### 7.1 主线

1. 端侧算力：`RB3 Gen2/QCS6490` 与 `Orin NX` 双路线评估
2. 相机：`RGB` 简配置主线
3. 音频：自研麦阵主线
4. 底盘：自研轮式底盘主线
5. 储物仓：能力保留、机构重构主线

### 7.2 备线

1. `RK3588` 作为成本下探与供应备线
2. 立体深度模块作为 Alpha / EVT 验证加速备线
3. 智能家居与穿戴按白名单扩展备线

### 7.3 观察线

1. `Jetson Thor / T5000`
2. 更重的端侧多模态大模型路线
3. 更复杂的深度传感器路线

## 8. `G2` 前必须完成的验证

为避免选型文档只停留在纸面，当前建议在 `G2` 前至少完成 7 项验证：

1. `RB3 Gen2/QCS6490` 与 `Orin NX` 的功耗、热设计和模型迁移对比
2. `RGB` 主线在关键家庭场景下的深度与障碍感知验证
3. 自研麦阵在整机结构内的远场识别与回声验证
4. 自研底盘与 `R1/R2` 执行栈的接口稳定性验证
5. 储物仓机构重构后的重量、可靠性和防夹手验证
6. 穿戴 / BLE / 智能家居输入链的稳定性验证
7. `BOM`、供应与交期的第一版对比表

## 9. 本轮联网核实的官方输入

本轮已优先参考官方资料，用于确认路线是否仍具备现实基础：

1. `NVIDIA Jetson` 官方模块页、生命周期页与 `JetPack 7` / `Thor` 发布信息
2. `Qualcomm Dragonwing RB3 Gen 2` 官方产品简报与 `AI Hub` 支持信息
3. `Rockchip RK3588` 官方产品页
4. `Intel RealSense D455 / D457` 官方产品页

说明：

- 上述资料只用于判断路线是否继续有效，不等于当前就完成了最终器件定型。
- 最终定型仍需要在 `G2` 前补齐价格、交期、板级获取、热设计和实测验证。

## 10. 当前待你审核的 5 点

1. 是否接受当前把端侧算力主线收敛为 `RB3 Gen2/QCS6490` 与 `Orin NX` 双路线评估。
2. 是否接受把 `RGB` 简配置 + 自研深度估计作为量产主线，把立体深度模块只保留在 Alpha / EVT 验证加速路线。
3. 是否接受继续保持“自研麦阵 + 自研底盘 + 外采穿戴生态”的总体组合。
4. 是否接受把 `RK3588` 定位为成本与供应备线，而不是一代默认主线。
5. 是否接受把伴生系统对本体的约束明确限定为“接口与职责约束”，而不是器件级反向锁定。
