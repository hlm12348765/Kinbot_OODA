# Kinbot_OODA 术语表

## 1. 目的

本文件用于统一项目术语，避免同一名词在不同文档里含义漂移。

规则：

- 尽量遵守智驾、机器人、具身智能和软件工程里已有的约定俗成用法
- 如果项目必须引入自定义术语，必须同时给出清晰定义
- 同一术语在不同文档中不得出现互相冲突的含义

## 2. 核心术语

### `World State`

定义：

- 机器人当前用于决策、执行和审计的统一结构化状态平面

包含：

- 当前人、物、空间、任务、风险、权限、健康和外部联动状态

不包含：

- 对未来环境动态的生成式预测能力

说明：

- 本项目当前文档中的认知中枢统一称为 `World State` 或 `World State Store`

### `World Model`

定义：

- 用于预测、生成、模拟或想象未来环境、任务或交互动态的模型

典型用途：

- 反事实推演
- 模拟与仿真
- 未来状态预测
- 规划前瞻

说明：

- 如果后续项目引入这类能力，可在 `Orient` 层作为可选预测模块加入
- `World Model` 不是当前“世界状态结构”的同义词

### `World Event`

定义：

- 从感知、任务或外部服务进入系统的结构化事件

### `DecisionContextSnapshot`

定义：

- 给一次具体决策使用的运行时快照

说明：

- 它是从 `World State` 投影出来的当前视图，而不是长期数据库

### `RoleBinding`

定义：

- 某个自然人在当前家庭中的角色和权限绑定关系

示例：

- 老人本人
- 子女
- 保姆
- 访客

### `Caregiver Mode`

定义：

- 面向保姆或照护者的协同工作模式

说明：

- 它不是一个人，而是一组针对 `caregiver` 角色启用的任务权限模板和交互策略

### `MedicationAsset`

定义：

- 被系统跟踪的药物实体，包括药品本身、存储位置、数量、有效期、处方引用和紧急使用规则

### `ServiceLink`

定义：

- 机器人与家庭外部服务主体之间的结构化联动对象

示例：

- 家属
- 社区
- 物业
- 医生平台
- 互联网医院
- 药店
- 配送服务
