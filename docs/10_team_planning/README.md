本目录用于收纳围绕 Kinbot 全生命周期推进所需的团队组织、岗位编制、阶段扩编、职责边界和协作机制提案。

它不替代系统架构、总体方案和技术路线文档，而是承接这些文档对组织能力的反推结果。

当前目录角色：

- 将系统架构和总体方案转译为团队结构提案
- 约束 `PDCP -> P2 -> 量产预备` 各阶段需要补齐的组织能力
- 为后续招聘优先级、到岗节奏、岗位说明、团队协作接口和 owner 分配提供输入
- 作为 `input/01_candidate_resume/` 候选人评估线程的组织能力基线输入

当前文档索引：

- `01_development_team_proposal.md`：开发团队提案。核心主题：基于当前架构与总体方案，定义 Kinbot 在量产预备前所需的 AI-native Team、人与 AI 协作机制、团队结构、人数规模、阶段扩编建议，以及 Step 43 吸收后的组织押注顺序与招聘节奏原则。
- `02_recruitment_requirements_optimized.csv`：招聘需求优化表。核心主题：把原始招聘需求统一成同一模板，并按当前团队规划基线维护岗位 `JD`、招聘优先级、期望到岗月份、优化建议，以及“为什么这个岗位押这个方向”这一层的方向选择论证，覆盖产品竞争力、架构理念、技术路线与市场 / 财务成功。

配套输入目录：

- `../../input/01_candidate_resume/README.md`：候选人简历输入目录说明。核心主题：候选人资料组织方式，以及与团队规划基线的配套使用边界。
