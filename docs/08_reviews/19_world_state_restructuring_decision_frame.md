## 1. 文档定位

本文档用于承接家庭共居智能体革新路线的 `Phase 3`。

它不直接重写 `World State` 主表，也不直接宣布 `9 -> 7` 成立，而是回答以下 `4` 个问题：

1. `Phase 2` 完成后，状态模型与数据模型到底还剩什么关键决策没有完成；
2. 当前应比较哪些候选路线，而不是继续模糊讨论；
3. 每条路线的收益、代价、触发条件与风险分别是什么；
4. 用户在这一轮审阅时，真正需要拍板的最小问题集是什么。

## 2. 当前阶段判断

到 `Phase 2` 结束时，当前主线已经完成：

1. 总架构从旧 `OODA` 总中心上抬为“家庭共居智能体”；
2. 运行时层改写为多执行范式表达；
3. `World State` 被明确定位为多执行范式共享的状态平面。

但以下事项仍未冻结：

1. 现有 `9` 类一级实体是否应重组为 `7` 类；
2. `CareRelationship / CareEvent` 是否应成为正式一级实体；
3. `V1` 的实际能力闭环，是否已经需要承担足够大的结构重组成本。

因此，`Phase 3` 的核心不是“立即改表”，而是“先完成状态模型与数据模型的正式决策”。

## 3. 当前决策问题重写

当前真正要回答的问题，不是：

**“`World State 9 -> 7` 看起来是不是更优雅？”**

而是：

**“在当前 `V1` 产品边界、模块并行设计和评审节奏下，Kinbot 是否已经具备承担 `World State` 主表重组成本的必要性与时机？”**

这会直接决定：

1. `Phase 3` 是应先做扩展层治理，还是直接重组主表；
2. 当前结构变化是帮助主线降复杂度，还是反而制造新的迁移成本；
3. 下游 `PDCP / KBT-32 / KBT-33` 文档是否会被迫再次大面积改写。

## 4. 当前必须遵守的约束

本轮评审必须同时满足以下约束：

1. 不得把 `provisional` 判断伪装成已冻结事实；
2. 不得为了结构美观而提前扩大迁移范围；
3. 不得破坏 `Phase 2` 已建立的多执行范式、共享状态平面和跨范式审批硬边界；
4. 不得让当前架构重新变复杂。

针对第 `4` 点，本轮明确追问一次：

**现在的架构是不是太复杂了？**

当前判断是：**如果在 `V1` 支撑关系和验证口径都还没写清之前，就直接重组 `World State` 主表，复杂度大概率会上升。**

## 5. 候选路线

### 5.1 路线 A：保留 `9` 类骨架 + 关系 / 事件扩展层

定义：

- 当前继续保留 `Person / RoleBinding / Household / Place / Object / HealthProfile / MedicationAsset / Task / RiskEvent` 这 `9` 类一级实体；
- 通过扩展层方式引入“关系视图”和“事件视图”，而不是立刻改主表；
- 关系与事件先作为跨实体治理层、聚合层和评审层存在。

在 `Step 47` 第 `7` 条吸收后，当前对路线 A 的判断已下调为：

1. 它可以作为对照路线，帮助说明为何不应继续停留在旧 `9` 实体模型；
2. 它不再适合作为当前主方向；
3. 它的主要价值在于暴露旧模型与新语义之间的冲突，而不是成为正式目标模型。

### 5.2 路线 B：直接执行 `World State 9 -> 7`

定义：

- 当前直接启动一级实体重组；
- 同步决定 `CareRelationship / CareEvent` 的正式地位；
- 把现有若干实体做合并、抽象或下沉。

这条路线的当前优势，在 `Step 47` 中已被进一步明确为：

1. 与已冻结原则层和运行时层保持一致；
2. 消除 `HealthProfile / MedicationAsset` 这类天然可合并的冗余一级实体；
3. 通过 `CareRelationship / CareEvent` 让数据模型与“家庭共居智能体”北极星保持语义一致；
4. 允许以“完整模型 + V1 最小激活子集”的方式推进，而不是继续维持上下不一致的过渡态。

## 6. 对比判断

| 维度 | 路线 A：`9` 骨架 + 扩展层 | 路线 B：直接 `9 -> 7` | 当前判断 |
| --- | --- | --- | --- |
| 与原则层 / 运行时层一致性 | 低 | 高 | `B` 更强 |
| 对结构冗余的消除力度 | 低 | 高 | `B` 更强 |
| 当前文档迁移代价 | 低 | 中到高 | `A` 更低 |
| 对长期竞争力的支撑 | 中低 | 高 | `B` 更强 |
| 对当前复杂度的长期压缩 | 低 | 高 | `B` 更强 |
| 当前推荐 | 否 | 是 | `B` 升为主方向 |

## 7. 当前推荐

在吸收 `Step 47` 第 `7` 条之后，当前推荐已调整为：

**路线 B 作为当前目标模型方向；路线 A 退出主决策位，仅保留为对照与风险说明。**

但这不等于“现在立刻做完整详细设计”。当前更准确的表达是：

**先冻结正确的七实体概念模型，再决定 `V1` 运行时只激活其中哪些子集。**

当前仍保持 `provisional` 的判断包括：

1. 七实体各自的完整字段边界；
2. `CareRelationship` 的最小 `V1` 激活子集；
3. `CareEvent` 的事件分类全集；
4. 从当前 `9` 骨架到 `7` 目标模型的迁移顺序。

## 8. 当前审阅状态

`Step 47` 已明确接受以下事项：

1. `Phase 2` 可以判定为“实质完成，允许收口”；
2. 当前对外正式口径已统一为“家庭共居智能体 + 多执行范式”；
3. `Phase 3` 的起点是单独评审 `World State 9 -> 7` 与“关系 / 事件扩展层”两条候选路线。

因此，本轮剩余真正需要继续审阅的重点，不再是“要不要进入 `Phase 3`”，而是：

1. 是否接受路线 B 升为当前目标模型方向；
2. 是否接受“完整模型正确，`V1` 子集激活”的推进方式；
3. 是否接受 `CareEvent` 与 `Task` 的边界规则。

## 9. 下一步动作

若本轮审阅接受本文框架，下一步应按以下顺序推进：

1. 明确七实体目标模型，见 `docs/08_reviews/21_seven_entity_world_state_target_model.md`；
2. 将路线 A 降为对照路线，仅保留其风险说明价值；
3. 在 `V1` 维度定义七实体中的最小激活子集；
4. 把 `CareEvent` 与 `Task` 的边界规则写成正式约束。
