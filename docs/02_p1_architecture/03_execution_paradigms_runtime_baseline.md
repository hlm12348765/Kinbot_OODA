## 1. 文档定位

本文档是 Kinbot 当前主线的运行时语法主文档，只回答 `4` 个问题：

1. 当前运行时包含哪 `4` 类执行范式；
2. 离散决策范式内部如何继续使用 `R1-R4`；
3. 当前执行范式协调机制依赖哪些输入与切换规则；
4. 各执行范式当前能做什么、不能越过什么边界。

背景、决策来路和“为什么从单一 `OODA` 走向多执行范式”的解释，统一放在 `14_family_co_living_agent_paradigm.md`。

## 2. 四类执行范式

### 2.1 离散决策范式

离散决策范式承接 `OODA` 的运行时语法，用于明确感知、认知、决策、动作和监督之间的离散阶段关系。

它适用于：

1. 需要显式审批和显式状态转移的动作；
2. 需要清晰责任边界和审计链的流程；
3. 需要把 `R1-R4` 分层执行环显式建模的场景。

### 2.2 连续流式范式

连续流式范式用于承接更前瞻的具身智能路线，例如：

1. 局部感知到动作的连续策略；
2. `VLA / 局部端到端`；
3. `Orient + Decide` 的融合表达。

它不要求显式分解为固定环节，但不能绕开系统级硬边界。

### 2.3 事件驱动范式

事件驱动范式用于承接跨尺度照护事件，包括：

1. 风险事件
2. 健康事件
3. 关系影响事件
4. 服务升级事件

其关键不是持续运行主环，而是：

`事件触发 -> 状态更新 -> 响应编排 -> 回写长期状态`

### 2.4 长周期演化范式

长周期演化范式用于承接长期关系形成、习惯学习、提醒策略优化和长期记忆治理。

它强调：

1. 长期可信
2. 长期克制
3. 长期可解释
4. 长期可恢复

## 3. 离散决策范式中的 `R1-R4`

当前确认把离散决策范式内部的执行环继续收敛为 `4` 类：

| 执行环 | 时间尺度 | 空间尺度 | 主要目标 | 典型能力 |
| --- | --- | --- | --- | --- |
| `R1 反射环` | 毫秒到百毫秒 | 机体周边、局部接触空间 | 立即规避危险 | 急停、避障、限速、夹手保护 |
| `R2 执行环` | `0.5` 秒到数秒 | 人附近、房间内局部空间 | 稳定执行当前动作 | 局部导航、到人确认、仓门执行、动作监督 |
| `R3 任务环` | 数秒到数分钟 | 房间到全屋 | 推进任务闭环 | 找人、送药、提醒升级、异常确认、回充仲裁 |
| `R4 关系与服务环` | 小时到天 | 家庭长期运行空间 | 优化长期服务 | 习惯学习、提醒策略、长期记忆治理、服务编排 |

## 4. 执行范式协调机制

当前仍保留一个显式的执行范式协调机制，用于决定当前以哪个执行范式、哪类执行环为主导。

### 4.1 协调输入

当前第一版继续收敛为 `6` 类输入：

1. `risk_level`
2. `novelty_level`
3. `confidence_level`
4. `task_deadline`
5. `resource_state`
6. `human_context`

### 4.2 当前切换规则

当前继续保留 `7` 条切换规则，但其语义已改写为“执行范式 / 执行环切换规则”：

1. 检测到碰撞、跌落、夹手等高风险信号时，任何高层范式都必须被 `R1` 抢占。
2. 到人确认、开仓、递送等已批准动作进入执行时，由 `R2` 主导。
3. 找人、送药、异常升级、回充仲裁等跨步骤任务，由 `R3` 主导。
4. 纪念日、习惯学习、提醒优化、长期服务策略调整，由 `R4` 或长周期演化范式主导。
5. 夜间、卫生间、身份不确定等高敏感上下文，会压低主动交互和大范围移动的尺度。
6. 网络断开时，服务升级与长周期策略能力要收缩，尽量回退到端侧可闭环范围。
7. 关键传感器失效且已无法识别障碍物时，所有涉及运动的执行环都必须中止。

## 5. 各执行范式边界

| 执行范式 | 当前主要作用 | 当前允许 | 当前不允许 |
| --- | --- | --- | --- |
| 离散决策范式 | 承接显式状态转移、显式审批、显式责任链的任务推进 | 使用 `OODA` 四阶段语法；按 `R1-R4` 分层组织；与事件驱动、长周期演化协同 | 把离散阶段写成所有场景都必须逐拍展开的固定流水线 |
| 连续流式范式 | 承接局部高频控制、局部端到端链路和 `Orient + Decide` 融合 | 局部端到端化；局部连续策略；在受控范围内减少显式阶段切换 | 绕开安全、授权、恢复和审计；替代全局运行时；以黑箱模型替代责任边界 |
| 事件驱动范式 | 承接风险、健康、服务升级和关系影响等跨尺度事件 | 事件触发状态更新；唤起离散执行环；直接触发服务接力；回写长期状态 | 把事件流写成与状态平面脱节的独立系统；跳过必要确认和升级链 |
| 长周期演化范式 | 承接习惯学习、提醒优化、关系形成和长期记忆治理 | 长周期策略修正；长期可信、克制、可解释的优化；和 `R4` 配合 | 追求短时最优；把未经验证的长期推断直接下放为即时动作 |

## 6. 附录 A：`OODA` 四阶段简记

`OODA` 在当前主线中只作为离散决策范式的运行时语法使用，可按下列简记理解：

1. `Observe`：围绕当前尺度组织采样、补采与状态确认。
2. `Orient`：完成认知评价、风险判断、尺度选择和必要的切环判断。
3. `Decide`：形成当前尺度下的动作或升级选择，并可在允许范围内与 `Orient` 融合。
4. `Act`：执行动作、监督结果、处理失败与回退，并把结果回写上层状态。
